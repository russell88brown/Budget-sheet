# Plan Checklist

Use this only when writing or reviewing `sprint-plan.md`.

## Required Sections

- `Metadata`
- `Objective`
- `Scope In`
- `Scope Out`
- `Task Checklist`
- `Acceptance Criteria`
- `Constraints`
- `Definition Of Done`
- `Risks And Mitigations`

## Validation Rules

- Every acceptance criterion is testable.
- Every task maps to one or more acceptance criteria.
- Scope out is explicit to avoid hidden work.
- Risks include mitigation, not only problem statements.
