# Review And Retro Checklist

Use this only during PR review and retro writing.

## Review Focus

- Acceptance criteria coverage
- Missing tests or weak evidence
- Behavioral regressions
- Documentation drift

## Retro Output Rules

- Capture wins and failures separately.
- Include root causes tied to process/system.
- Write action items with owner and due date.
- Prefer preventive actions over generic reminders.
